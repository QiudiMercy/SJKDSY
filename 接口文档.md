# 《带我玩》Web游戏 API 接口文档

本文档定义了前后端交互的 RESTful API 规范，所有返回数据统一遵循以下 JSON 基础结构：

```json
{
  "code": 200,      // 业务状态码：200 成功，400 参数错误，500 服务器错误
  "msg": "success", // 提示信息
  "data": {}        // 具体载荷数据
}
```

---

## 模块一：游戏状态与全局管理 (Game)

### 1.1 初始化新游戏

- **路径**: `/api/game/start`
- **方法**: `POST`
- **描述**: 点击开始游戏时调用，创建一局新游戏并返回初始状态。
- **请求参数**: 无
- **返回参数 (data)**:
  ```json
  {
    "game_uid": "g_123456789",
    "init_state": {
      "time": "08:00",           // 游戏内当前时间
      "money": 1000,             // 初始资金（元）
      "stamina": 100,            // 体力值 (0-100)
      "mood": 100,               // 心情值 (0-100)
      "location": {
        "name": "成都东站",
        "lng": 104.148151,
        "lat": 30.634674
      }
    }
  }
  ```

### 1.2 获取当前状态

- **路径**: `/api/game/state`
- **方法**: `GET`
- **描述**: 获取当前游戏各项数值面板状态。
- **请求参数 (Query)**:
  - `game_uid` (String): 当前游戏UID
- **返回参数 (data)**: 与 `init_state` 结构相同，并增加 `is_game_over` 字段。

### 1.3 游戏结算

- **路径**: `/api/game/settle`
- **方法**: `POST`
- **描述**: 当触发结束条件（资金0、时间22:00、体力心情0）时调用，返回最终评分和历史游玩路线。
- **请求参数 (Body)**:
  - `game_uid` (String): 当前游戏UID
- **返回参数 (data)**:
  ```json
  {
    "score": 95,
    "evaluation": "小爱度过了非常充实的一天，你是一个合格的导游！",
    "route_summary": ["成都东站", "太古里", "宽窄巷子"]
  }
  ```

## 模块二：AI 聊天互动 (Chat)

### 2.1 发送对话消息

- **路径**: `/api/chat/send`
- **方法**: `POST`
- **描述**: 向小爱发送消息，后端 Agent 判定后返回消息并（可选）更新系统状态（系统裁定）。*(注：如果是流式输出，此接口建议转换为 SSE 协议或 WebSocket)*
- **请求参数 (Body)**:
  - `game_uid` (String)
  - `content` (String): 用户发送的消息文本
- **返回参数 (data)**:
  ```json
  {
    "reply": "好呀，我们现在去哪？",      // Agent 的文本回复
    "is_system_reply": true,            // 系统裁定是否更新状态值
    "system_reply": "用户在小爱生气的时候关心，心情值+1", // 如果更新状态值，系统给出裁定理由
    "new_status": {
        "mod": 80                      // 心情值更新到80
    }
  }
  ```

### 2.2 获取历史聊天记录

- **路径**: `/api/chat/history`
- **方法**: `GET`
- **描述**: 获取单局游戏的完整历史对话。
- **请求参数 (Query)**:
  - `game_uid` (String)
- **返回参数 (data)**:
  ```json
  {
    "messages": [
      {
        "role": "user",
        "content": "你到哪啦？",
        "timestamp": "08:10"
      },
      {
        "role": "xiaoai",
        "content": "我刚出站呢，成都天气真好！",
        "timestamp": "08:11"
      },
      {
        "role": "system",
        "content": "你们刚刚吃完火锅，饱食度+1"， // 示例
        "timestamp": "08:15",
      },
    ]
  }
  ```

## 模块三：地图与行动决策 (Action)

### 3.1 搜索附近的或指定的 POI

- **路径**: `/api/poi/search`
- **方法**: `GET`
- **描述**: 依据关键词检索成都本地的地点信息。
- **请求参数 (Query)**:
  - `keyword` (String): 搜索关键词，不填则默认请求附近的推荐点位
  - `lng` (Float): 当前经度
  - `lat` (Float): 当前纬度
- **返回参数 (data)**:
  ```json
  {
    "poi_list": [
      {
        "poi_uid": "p_001",
        "name": "成都远洋太古里", // 信息还可以增加，具体看能找到什么样的数据
        "type": "商业街",
        "distance": "10km",
        "lng": 104.091122,
        "lat": 30.658688
      }
    ]
  }
  ```

### 3.2 路线规划预估

- **路径**: `/api/action/route`
- **方法**: `POST`
- **描述**: 选择一个目的地，返回不同出行方式（打车/地铁/步行）的耗时与花费预估。
- **请求参数 (Body)**:
  - `target_poi_uid` (String): 目的地UID
  - `current_lng` (Float)
  - `current_lat` (Float)
- **返回参数 (data)**:
  ```json
  {
    "routes": [
      {
        "method": "taxi",
        "label": "打车",
        "cost_money": 35,        // 预估花费35元
        "cost_time_min": 25,     // 预估耗时25分钟
        "consume_stamina": -5    // 打车稍微扣除一点体力
      },
      {
        "method": "subway",
        "label": "地铁",
        "cost_money": 4,
        "cost_time_min": 40,
        "consume_stamina": -15
      },
      {
        "moethod": "walk",
        "label": "步行",
        "cost_money": 0,
        "cost_time_min": 60,
        "consume_stamina": -30
      }
    ]
  }
  ```

## 模块四：战绩与历史 (Records)

### 4.1 获取战绩列表

- **路径**: `/api/records/list`
- **方法**: `GET`
- **描述**: 首页展示过往游戏的得分列表。
- **请求参数**: 无 或 分页参数 `page`, `limit`
- **返回参数 (data)**:
  ```json
  {
    "records": [
      {
        "game_uid": "g_123456789",
        "start_time": "2023-10-31 10:00:00", // 详细内容后续商定
        "score": 95,
        "remain_money": 120
      }
    ]
  }
  ```
